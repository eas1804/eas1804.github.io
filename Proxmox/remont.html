Устранение неполадок

Выяснить серейный номер дефектного диска

Допустим, мы видим проблему "too many errors" 

zpool status
....
wwn-0x5000039fc3ebff79            FAULTED      3   844     0  too many errors

Выяснеяем сериный номер

lsblk -o NAME,SERIAL,WWN | grep 5000039fc3ebff79

Замена коротких имен

Было 
            
        NAME        STATE     READ WRITE CKSUM
        pve-pool    ONLINE       0     0     0
          raidz2-0  ONLINE       0     0     0
            sdb     ONLINE       0     0     0
            sdc     ONLINE       0     0     0

eСтало

Использование имен wwn- (World Wide Name) — это самый надежный способ идентификации дисков 

            wwn-0x5000039fe7d1d033  ONLINE       0     0     0
            wwn-0x5000c50064ecbc78  ONLINE       0     0     0

Экспортируйте пул (отмонтирует его):

zpool export pve-pool

Импортируйте обратно, указав путь к ID:

zpool import -d /dev/disk/by-id pve-pool
