Управление службами

  • Статус службы sc query ZvitGrp2

    Расшифровка поля STATE:

    1. STOPPED — Служба остановлена.
    2. START_PENDING — Служба запускается.
    3. STOP_PENDING — Служба останавливается.
    4. RUNNING — Служба работает.
    5. CONTINUE_PENDING — Служба возобновляется.
    6. PAUSE_PENDING — Служба приостанавливается.
    7. PAUSED — Служба приостановлена.
  • Запустить/останвоить службу net start ZvitGrp2 net stop "Zabbix Agent"
  • Поведение при включении компьютера

    sc config MSiSCSI start= auto

    Варианты start=:

    1. auto — Автоматический запуск службы при загрузке системы.

      Служба запускается каждый раз, когда компьютер включается.

    2. demand — Ручной запуск (по запросу).

      Служба не запускается автоматически, но ее можно запустить вручную через команды, интерфейс управления службами или приложение, которое ее использует.

    3. disabled — Служба отключена.

      Служба не может быть запущена вручную или автоматически, пока не будет снова включена.

    4. delayed-auto — Автоматический запуск с задержкой.

      Служба запускается после завершения всех стандартных автоматических служб. Это полезно для оптимизации времени загрузки системы.

    5. boot — Служба запускается на этапе загрузки системы.

      Обычно используется для низкоуровневых драйверов и служб, требующихся на ранних этапах загрузки. (Этот параметр применим только для драйверов, а не для стандартных служб.)

    6. system — Запуск службы системой (также для драйверов).

      Службы или драйверы, загружаемые непосредственно ядром. (Этот вариант аналогично относится к драйверам и используется редко.)
